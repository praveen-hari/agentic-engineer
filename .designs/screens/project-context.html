How it works: Every workflow action (spec generated, task completed, approval given) is recorded in events.jsonl and the current state is updated in workflow.json. Both are committed with each task's code changes. When a teammate pulls the branch, the extension reads workflow.json and reconstructs the full UI state — including which stage is active, which tasks are done, and what approvals are pending.
Team Handoff
Resume after restart: Extension reads workflow.json on activation — picks up exactly where you left off
Switch branches: Each branch can have its own .codestudio/workflows/current/ — switching branches switches workflow state
Team member joins: Pull the branch → open Code Studio → extension loads the full engineering state automatically
Approval attribution: Each approval records who approved and when — visible in the audit log
PR review: Reviewers can see the full engineering story — spec, plan, decisions, test evidence — in the .codestudio/ directory
