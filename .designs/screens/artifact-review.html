Spec: Stripe Payment Integration

Objective

Add Stripe payment processing to the checkout flow. Users can pay with credit/debit cards. Payment completes in <3s, handles failures gracefully, and follows PCI compliance best practices (no card data touches our server).

Tech Stack

  • Stripe.js v3 (client-side) — stripe.com/docs/js
  • stripe-node v14 (server-side) — stripe.com/docs/api
  • Zod for input validation (consistent with existing patterns)

Commands

Test: npm test -- --grep "payment"
Dev: npm run dev
Build: npm run build

Testing Strategy

  • Unit: Payment validation, amount calculation, error mapping
  • Integration: Stripe API mock with test keys
  • E2E: Full checkout flow with Playwright

Boundaries

Always Validate amounts server-side · Use Stripe.js for card collection (never touch raw card data)
Ask first Storing payment metadata · Adding webhook endpoints · Retry logic for failed payments
Never Log full card numbers · Store card data on our server · Disable HTTPS · Expose Stripe secret key

Success Criteria

Payment intent created successfully via API
Card validation errors shown to user with clear messages
Failed payments handled with retry option
Payment confirmation page shows receipt details
All amounts in cents (integer) — no floating point
